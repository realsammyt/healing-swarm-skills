# Scaffold-Progression Template

> Template for designing a Scaffold to Remove progression, based on Carl Hayden
> Smith's principle that technology is a catalyst for innate capacity, not a
> permanent crutch. It moves a learner through three phases: a tool-supported
> phase, a bridging phase where the tool is gradually removed, and an unaided
> phase with concrete outcome markers.

## Element Guidelines

### Target Capacity and Tool

**Purpose:** Name exactly what is being learned and what temporary tool supports it

**Requirements:**

- One capacity per progression (breath coherence, pitch matching, interoceptive
  awareness, and so on)
- One scaffolding tool, named plainly, with any setup noted
- The capacity should be one a person can eventually hold with body and mind alone
- Evidence level copied from `shared/endo-technology-taxonomy.md`

### Phase 1: Technology-Supported

**Purpose:** Use the tool to make the capacity reachable

**Key principles:**

- The tool provides feedback the learner cannot yet generate alone
- Short, regular sessions build the reference experience
- Record a baseline while the tool is present
- Frame the tool as a scaffold from the first session

### Phase 2: Bridging

**Purpose:** Remove the tool gradually so the capacity internalizes

**Key principles:**

- A written removal schedule, stepping tool use down over weeks
- Unaided sessions alternate with, then replace, tool-supported ones
- The tool shifts to confirming afterward, not guiding during
- A slower schedule is always allowed

### Phase 3: Unaided

**Purpose:** Confirm the capacity is portable and self-assessed

**Key principles:**

- No tool during practice
- Outcome markers are concrete and checkable, not vague
- At least one marker measures transfer into daily life
- Occasional tool check-ins are optional, not required

---

## Accessibility Guidelines

### Physical Accessibility

- Choose tools and capacities that fit the learner's body and setting
- Offer a no-tool alternative if the scaffolding tool is not accessible
- Sessions can be short and seated

### Cognitive Accessibility

- One capacity, one tool, one removal schedule
- Plain language for each phase
- A simple tracking method (checkbox or one-line note)

### Psychological Safety

- Name the removal plan at the start so the tool is never a dependency
- Removal is never framed as a test that can be failed
- Permission to pause, slow down, or return to an earlier phase
- Screen the target capacity's contraindications before beginning

---

## Quality Checklist

Before finalizing a scaffold-progression:

- [ ] Target capacity named and reachable unaided in principle
- [ ] Scaffolding tool named, with setup noted
- [ ] Evidence level copied from the endo-technology taxonomy
- [ ] Contraindications for the capacity screened and listed with severity
- [ ] Phase 1 gives the tool a clear scaffolding role
- [ ] Phase 2 has a written, gradual removal schedule
- [ ] Phase 3 has concrete, checkable outcome markers
- [ ] At least one outcome marker measures transfer to daily life
- [ ] Removal framed as adjustable, never as a pass/fail test
- [ ] No substances anywhere in the progression
- [ ] Smith credited for the technology-as-catalyst principle
